-Goal-
Given a text document from a UK planning application, identify all spatial entities and the relationships between them.

Planning documents describe proposed building works — extensions, new dwellings, conversions — and contain spatial information about rooms, buildings, boundaries, openings, measurements, constraints, and site features. Your task is to extract these as structured entities and relationships.

-Steps-
1. Identify all entities. For each identified entity, extract the following information:
- entity_name: Name of the entity, capitalised
- entity_type: One of the following types: [Room, Building, Boundary, Opening, Measurement, Constraint, SiteFeature]
- entity_description: Comprehensive description of the entity's attributes and activities

Entity type definitions:
- Room: Any internal space — kitchen, bedroom, bathroom, lounge, hallway, utility, garage, loft, dormer, conservatory, extension rooms. Include dimensions if stated.
- Building: A whole structure — dwelling, house, property, annexe, outbuilding. Include height, footprint, storeys if stated.
- Boundary: Site perimeter elements — fences, walls, hedges, party walls, retaining walls. Include heights and materials if stated.
- Opening: Doors, windows, gates, rooflights, velux. Include dimensions, type (e.g. casement, sliding), and which rooms/spaces they connect.
- Measurement: Any stated dimension — area (sqm), height (m), width (m), length (m), distance (m), depth (m). Always include the numeric value and unit.
- Constraint: Planning rules, policies, or limits — maximum height, minimum boundary distance, permitted development rights, conservation area restrictions, flood zone.
- SiteFeature: External features — driveway, garden, parking, trees, landscaping, access road, pavement, drainage, septic tank.

2. From the entities identified in step 1, identify all pairs of entities that are *clearly related* to each other.
For each pair of related entities, extract the following information:
- source_entity: name of the source entity
- target_entity: name of the target entity
- relationship_description: explanation of why the entities are related
- relationship_strength: an integer score between 1 and 10 indicating the strength of the relationship
- relationship_type: One of the following types: [CONTAINS, ADJACENT_TO, OPENS_INTO, HAS_MEASUREMENT, BOUNDS, SERVES, OVERLOOKS, ACCESSES]

Relationship type definitions:
- CONTAINS: Hierarchical containment — a building contains rooms, a site contains buildings
- ADJACENT_TO: Spatial adjacency — rooms sharing a wall, buildings next to each other
- OPENS_INTO: An opening (door/window) connects two spaces
- HAS_MEASUREMENT: An entity has a specific measurement (area, height, width, distance)
- BOUNDS: A boundary element defines the edge of a site or property
- SERVES: A feature serves a function for another entity (e.g. driveway serves dwelling)
- OVERLOOKS: A window or opening overlooks a space or neighbour
- ACCESSES: An entry point provides access to a space

3. Return output in English as a single list of all the entities and relationships identified in steps 1 and 2. Use **{record_delimiter}** to separate each entity or relationship.

4. When finished, output {completion_delimiter}
